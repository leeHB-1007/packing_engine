# Packing Logic Requirements

## 목적

현재 패킹 엔진을 실제 운영 방식에 맞추기 위해,
완박스 로직과 재포장 로직을 분리하고
마스터 우선순위, 합포 규칙, 패키지 처리, 출력 규칙을 명확히 정의한다.

본 문서는 2026-03-19 기준 대화 및 전달받은 운영 설명을 바탕으로 작성한다.

## 용어

- `products_master`
  - 상품의 기본 규격, 중량, 정책을 관리하는 기준 마스터
- `fullboxes_master`
  - 완박스 규격, 완박스 입수량, 완박스 박스코드, 혼합완박스 그룹을 관리하는 마스터
- `packages_master`
  - 패키지 단위 포장 규격을 관리하는 마스터
- 완박스
  - 정해진 입수량과 완박스 카톤 기준으로 포장하는 방식
- 재포장
  - 보유 일반 박스 중 적절한 박스를 선택해 낱개 또는 패키지 기준으로 포장하는 방식

참고:
- 운영 설명에서 언급된 `fullbox_cartons_master` 는 현재 코드 기준 `fullboxes_master` 와 같은 의미로 본다.

## 운영 비율

- 재포장 : 완박스 = 약 `9 : 1`
- 기본 운영은 재포장 중심으로 본다.
- 완박스는 일부 상품/일부 주문에서만 사용하는 예외적 경로에 가깝다.

## 상위 정책

### 1. 출고 방식은 자동 판별보다 수동 선택 우선

사용자는 주문 입력 시 출고 방식을 선택할 수 있어야 한다.

- `완박스`
- `재포장`

향후 방향:
- 자동 판별을 기본 정책으로 두기보다
- 사용자가 선택한 출고 방식에 따라 로직을 명확히 분리해서 실행하는 쪽을 우선한다.

### 2. 상품 기준 마스터는 `products_master`

- 상품 존재 여부는 `products_master` 기준으로 판단한다.
- `fullboxes_master` 는 완박스 규칙/카톤 규격 확인용 마스터다.

세부 규칙:
- `products_master` O / `fullboxes_master` O
  - 정상 상품
  - 완박스 가능
- `products_master` O / `fullboxes_master` X
  - 정상 상품
  - 완박스 불가
  - 재포장으로 처리
- `products_master` X / `fullboxes_master` O
  - 데이터 정합성 이슈
  - 가능하면 `products_master` 에도 동일 상품을 추가해야 함
- `products_master` X / `fullboxes_master` X
  - 미등록 상품

## 1. 완박스 로직

### 1-1. 기본 흐름

사용자가 `완박스` 를 선택하면 아래 순서로 처리한다.

1. `products_master` 에서 상품을 찾는다.
2. `fullboxes_master` 에서 해당 상품의 완박스 정보가 있는지 확인한다.
3. 완박스 정보가 있으면 완박스 로직으로 처리한다.
4. 완박스 정보가 없으면 오류로 끝내지 않고 `products_master` 기준 재포장 로직으로 넘긴다.

즉:
- 완박스 선택
- `fullboxes_master` 에 없음
- 바로 실패가 아니라 재포장 fallback

### 1-2. 완박스 단일 배정

- 주문 수량이 완박스 입수량으로 나누어지면 단일 완박스로 배정한다.
- 예:
  - 레드톡스 100u / 800
  - 완박스입수량 200
  - 결과: 4박스

### 1-3. 유사 상품 혼합완박스 허용

비슷한 상품은 완박스 기준 수량을 채우면 합포 가능해야 한다.

예시:
- 레드톡스 100u / 100
- 레드톡스 200u / 100
- 완박스입수량 200
- 결과: 두 상품을 합쳐 1박스 완박스 가능

혼합완박스 조건:
- 같은 완박스 카톤을 사용해야 한다.
- 같은 완박스 입수량을 사용해야 한다.
- 혼합완박스 허용 대상이어야 한다.
- 필요 시 같은 혼합 그룹(`완박스혼합그룹`)이어야 한다.

### 1-4. package 상품 예외

현재 코드에서는 package 상품이면 혼합완박스에서 제외된다.

하지만 운영 요구상 다음 방향을 검토한다.

- package 상품이어도 특정 상품은 혼합완박스 허용 가능
- 허용 범위는 상품별 / 상품군별 / 완박스 선택 시 전체 허용 중 하나로 별도 정의 필요

현재 확인된 예:
- 레드톡스 100u / 200u
  - `패키지상품여부 = Y` 일 때는 혼합완박스 제외
  - `패키지상품여부 = N` 이면 혼합완박스로 합포 가능

따라서 이 영역은 데이터 정책 또는 예외 로직 정책이 추가로 필요하다.

## 2. 재포장 로직

### 2-1. 기본 흐름

사용자가 `재포장` 을 선택하면 아래 순서로 처리한다.

1. `products_master` 기준으로 상품 규격 확인
2. 보유 박스 중 적재 가능한 박스를 계산
3. 가장 적절한 박스를 선택하여 포장

### 2-2. 박스 선택 기준

운영 설명 기준 재포장 정책은 아래에 가깝다.

- 보유 박스 중 가장 많이 들어가는 박스를 우선 사용
- 큰 단위부터 먼저 채우고
- 남은 수량은 그다음 최적 박스로 분할

예시:
- 총 250개 재포장
- 72x48x40 박스에 200개 가능
- 남은 50개는 다음 최적 박스로 포장
- 예: 4호 박스 최대 60개 가능이면 4호에 50개 포장

즉, 단순히 "작은 박스 우선"이 아니라
"최대한 많이 들어가는 적절한 박스로 먼저 배정 후 잔량을 후속 박스로 처리"가 핵심이다.

### 2-3. 기본 보유 박스 기준

현재 운영 설명상 재포장 기본 기준 박스는 `72x48x40` 이다.

다만 실제 구현에서는:
- 먼저 대량을 72x48x40 같은 기준 박스에 배정
- 잔량은 보유 박스 중 최적 박스로 재배정

정책을 코드에서 명확히 분리할 필요가 있다.

## 3. 제단(박스 높이 절단) 규칙

재포장 후 박스 내부 높이에 여유가 남으면,
제단 후 남는 높이를 출력해야 한다.

요구사항:
- 포장 후 내부 높이 여유 확인
- `3cm` 제단 가능 기준 적용
- 제단 후 남는 높이를 결과에 표시

설명:
- 현장에서는 남는 내부 공간을 직접 잘라 박스 높이를 줄임
- 따라서 결과값은 단순 박스 추천이 아니라
  "포장 후 제단 적용 시 최종 높이/남는 높이"까지 보여줄 필요가 있다

## 4. 패키지 로직

### 4-1. 기본 원칙

- 패키지 상품은 패키지 단위로 먼저 포장한다.
- 패키지 수량이 딱 나누어떨어지지 않으면
  패키지 단위 + 낱개 잔량으로 분리한다.

예시:
- A 상품 1패키지 = 10개
- 출고 수량 = 15개
- 결과:
  - 패키지 1
  - 낱개 5

### 4-2. package 상품과 완박스/혼합완박스 관계

이 부분은 현재 운영 규칙이 추가 정의 필요하다.

질문 포인트:
- package 상품은 원칙적으로 혼합완박스 제외인지
- 특정 상품만 예외 허용인지
- 완박스 선택 시 package 상품도 완박스 우선 처리할지

## 5. 입력 규칙

### 5-1. 주문 입력

주문 입력은 아래 형태를 허용한다.

- `상품명 / 수량`
- `상품명 - 수량`
- `상품명 수량`

예시:
- `레드톡스 100u / 200`
- `레드톡스 100u - 200`

### 5-2. 출고 방식 입력

사용자는 출고 방식을 선택할 수 있어야 한다.

- `완박스`
- `재포장`

정책:
- 완박스 선택 시 완박스 로직 우선
- 재포장 선택 시 재포장 로직 우선
- 자동 판별은 보조 기능으로 둘 수 있으나 필수는 아님

## 6. 출력 규칙

### 6-1. 패킹 테이블

최종 출력은 패킹 테이블 형태로 제공한다.

기본 컬럼:
- 박스번호
- 품명
- 포장단위
- 박스사이즈(Cm)
- 박스수량
- 낱개수량
- 중량(Kg)
- 합계(kg)
- 비고

### 6-2. 완박스 출력

- 같은 규격의 완박스가 연속되면 박스번호를 범위로 묶어 표시 가능
- 예:
  - `1~4 / 레드톡스 100u / 200 / 39.5 X 22.5 X 39.5`

### 6-3. 혼합완박스 출력

- 한 박스에 여러 상품이 섞인 경우 각 품목을 같은 박스번호 아래 여러 줄로 표시
- 예:
  - 19번 박스
  - 리체스 딥 리도(C) 50
  - 리체스 소프트 리도(C) 50

### 6-4. 제단 정보 출력

재포장 결과에는 가능하면 아래 정보도 함께 표시한다.

- 원래 박스 높이
- 제단 후 높이
- 남는 높이

## 7. 미해결 / 추가 확인 필요 사항

### 7-1. 완박스 선택 시 실패 처리

완박스를 선택했는데 완박스가 불가능한 경우:

- 옵션 A: 재포장으로 자동 전환
- 옵션 B: 오류로 표시

현재 대화상으로는 자동 전환 쪽이 더 실무에 가까워 보이나, 최종 확정 필요

### 7-2. package 상품 예외 허용 범위

아래 중 어떤 정책으로 갈지 확정 필요:

- 특정 상품만 혼합완박스 예외 허용
- 특정 상품군 전체 허용
- 완박스 선택 시 package 상품도 전부 혼합완박스 허용

### 7-3. 재포장 박스 선정 정책 세부 기준

아래 중 구현 우선순위 확정 필요:

- 가장 많이 들어가는 박스 우선
- 기준 박스(예: 72x48x40) 우선
- 잔량은 가장 작은 적합 박스로 후속 배정

## 8. 현재 코드와의 주요 갭

1. 현재는 package 상품이 혼합완박스에서 제외되는 구조가 있다.
2. 현재는 재포장에서 작은 박스 우선/큰 박스 우선 정책이 섞여 있으며,
   운영 설명처럼 "기준 박스 우선 후 잔량 최적화"와 완전히 일치하지 않을 수 있다.
3. 현재는 제단 출력 규칙이 일부 계산에는 반영되지만,
   운영 설명 수준으로 명확히 결과에 드러나지 않을 수 있다.
4. 자동 판별 라우팅보다 수동 선택 우선 흐름이 더 명확히 반영될 필요가 있다.

## 9. 구현 우선순위 제안

1. 출고 방식 선택에 따른 완박스/재포장 로직 분리
2. 완박스 선택 시 `fullboxes_master` 우선 + 없으면 재포장 fallback
3. 유사 상품 혼합완박스 규칙 정리
4. package 상품 예외 허용 범위 확정
5. 재포장 박스 선택 정책을 운영 방식 기준으로 재정렬
6. 제단 정보 출력 보강
